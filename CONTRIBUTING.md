# Contributing Guidelines

This is an academic project for COE-506.

## How to contribute

- Fork the repo, make a branch, push, and open a PR.
- Describe what you changed and how you tested it (even a quick run).
- If you canâ€™t test on GPU, say so.

## What to work on

- Fix bugs or typos.
- Improve docs or examples.
- performance tweaks.

## Code style

- Python: short comments, readable lines.
- CUDA C/C++: check for errors, keep launch settings commented

## Testing

- Python: run the script/notebook cell you touched.
- CUDA: `make` and run once; mention profiling only if you did it.

## Academic integrity

- Use your own work; cite any external code or references.
- Contributions are under MIT (see `LICENSE`).

## Need help?

- Open a short issue describing the problem or question.
